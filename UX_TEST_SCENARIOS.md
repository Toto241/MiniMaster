# UX-Testszenarien für die Mini-Master Kind-App – Gerätekopplung

Diese Dokumentation listet die wichtigsten User Experience (UX) Testszenarien für die Gerätekopplungsfunktion der Kind-App auf. Sie dient als Grundlage für manuelle Tests und zur Überprüfung der Benutzerfreundlichkeit.

## Teil 1: Kopplungsprozess (`PairingScreen`)

| Testfall-ID | Szenario                                                                 | Wichtigste erwartete Ergebnisse                                                                                                                               |
| :---------- | :----------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| UX-CP-001   | **Erfolgreiche Kopplung (Happy Path)** mit gültigem Code.                | - Eingabe des Codes möglich.<br>- Klick auf "Gerät koppeln" Button.<br>- Ladeanzeige wird während der Validierung angezeigt.<br>- Nach Erfolg: Navigation zum `LockScreen`.<br>- Im `LockScreen`: Korrekte `childId` wird angezeigt.<br>- Der Kopplungscode wurde serverseitig gelöscht (nicht direkt in UI sichtbar, aber wichtiges Systemverhalten). |
| UX-CP-002   | **Ungültiger Kopplungscode** (nicht existent oder syntaktisch falsch).   | - Eingabe des Codes möglich.<br>- Klick auf "Gerät koppeln" Button.<br>- Ladeanzeige wird angezeigt.<br>- Fehlermeldung "Ungültiger Kopplungscode" (oder ähnlich, je nach String-Ressource) wird angezeigt.<br>- Verbleib im `PairingScreen`.<br>- Eingabefeld für Kopplungscode ist weiterhin aktiv und editierbar.<br>- Button "Gerät koppeln" ist wieder aktiv (nachdem Ladeanzeige verschwindet). |
| UX-CP-003   | **Abgelaufener Kopplungscode.**                                          | - Eingabe des Codes möglich.<br>- Klick auf "Gerät koppeln" Button.<br>- Ladeanzeige wird angezeigt.<br>- Fehlermeldung "Kopplungscode ist abgelaufen" (oder ähnlich) wird angezeigt.<br>- Verbleib im `PairingScreen`.<br>- Eingabefeld aktiv.<br>- Button wieder aktiv. |
| UX-CP-004   | **Netzwerkfehler während der Code-Validierung** (z.B. Flugmodus aktivieren). | - Eingabe des Codes möglich.<br>- Klick auf "Gerät koppeln" Button.<br>- Ladeanzeige wird angezeigt.<br>- Passende Fehlermeldung (z.B. "Netzwerkfehler", "Keine Internetverbindung" oder generischer Fallback "Ungültiger Kopplungscode", je nach Implementierung) wird angezeigt.<br>- Verbleib im `PairingScreen`.<br>- Button "Gerät koppeln" wird nach Fehlerbehandlung wieder reaktiviert. |
| UX-CP-005   | **Fehler beim Speichern der `childId`** nach erfolgreicher Validierung (Simulation, falls möglich, oder Überprüfung der Fehlermeldung). | - Eingabe eines gültigen Codes.<br>- Klick auf "Gerät koppeln" Button.<br>- Ladeanzeige wird angezeigt.<br>- Fehlermeldung "Fehler beim Speichern der Geräte-ID" (oder ähnlich) wird angezeigt.<br>- Verbleib im `PairingScreen`.<br>- Der Kopplungscode wurde serverseitig bereits gelöscht (wichtiges Systemverhalten). |
| UX-CP-006   | **Leere Eingabe für Kopplungscode** und Klick auf Button.                 | - Eingabefeld für Kopplungscode ist leer.<br>- Button "Gerät koppeln" ist deaktiviert.<br>- Keine Aktion bei versuchtem Klick (falls irgendwie möglich). |
| UX-CP-007   | **Mehrfache schnelle Klicks** auf den Kopplungs-Button mit gültigem Code.   | - Eingabe eines gültigen Codes.<br>- Mehrfache schnelle Klicks auf "Gerät koppeln".<br>- Nur ein Kopplungsversuch wird ausgelöst (Ladeanzeige erscheint nur einmal).<br>- Keine Abstürze oder unerwartetes Verhalten.<br>- Normale Prozessabwicklung (Erfolg oder Fehlschlag des einen Versuchs). |

## Teil 2: App-Start

| Testfall-ID | Szenario                                                                 | Wichtigste erwartete Ergebnisse                                                                                                                               |
| :---------- | :----------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| UX-AS-001   | **App-Start mit gespeicherter `childId`.**                                 | - App wird gestartet.<br>- Direkter Start in den `LockScreen`.<br>- Die korrekt gespeicherte `childId` wird auf dem `LockScreen` angezeigt.                         |
| UX-AS-002   | **App-Start ohne gespeicherte `childId`.**                                 | - App wird gestartet (z.B. nach Neuinstallation oder Löschen der App-Daten).<br>- Start in den `PairingScreen`.                                               |
| UX-AS-003   | **Fehler beim Lesen der `childId` aus DataStore** (Optional/Schwer manuell testbar ohne spezielle Test-Builds, die Fehler simulieren). | - App wird gestartet.<br>- Kein Absturz der App.<br>- Fallback-Verhalten: Start im `PairingScreen` (als ob keine `childId` vorhanden wäre).                      |

## Teil 3: Internationalisierung (i18n)

| Testfall-ID | Szenario                                                                        | Wichtigste erwartete Ergebnisse                                                                                                                                                                                                                                                                                                                                                                                          |
| :---------- | :------------------------------------------------------------------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| UX-I18N-001 | **Überprüfung der deutschen Lokalisierung.** (Gerätesprache auf Deutsch stellen)  | - Im `PairingScreen`: Titel ("Kopplungsbildschirm"), Label des Eingabefelds ("Kopplungscode eingeben"), Button-Text ("Gerät koppeln") sind korrekt auf Deutsch.<br>- Fehlermeldungen (z.B. "Ungültiger Kopplungscode.", "Kopplungscode ist abgelaufen.", "Fehler beim Speichern der Geräte-ID.") sind korrekt auf Deutsch.<br>- Im `LockScreen`: Die Nachricht ("App ist gesperrt. Kind-ID: %1$s") ist korrekt auf Deutsch. |
| UX-I18N-002 | **Überprüfung der französischen Lokalisierung.** (Gerätesprache auf Französisch stellen) | - Im `PairingScreen`: Titel, Label, Button-Text sind korrekt auf Französisch.<br>- Fehlermeldungen sind korrekt auf Französisch.<br>- Im `LockScreen`: Die Nachricht ist korrekt auf Französisch.                                                                                                                                                                                                             |
| UX-I18N-003 | **Überprüfung der chinesischen (vereinfacht) Lokalisierung.** (Gerätesprache auf Chinesisch, vereinfacht stellen) | - Im `PairingScreen`: Titel, Label, Button-Text sind korrekt auf Chinesisch.<br>- Fehlermeldungen sind korrekt auf Chinesisch.<br>- Im `LockScreen`: Die Nachricht ist korrekt auf Chinesisch.                                                                                                                                                                                                          |
| UX-I18N-004 | **Überprüfung des Fallbacks auf Englisch** bei nicht unterstützter Sprache. (Gerätesprache auf eine nicht übersetzte Sprache stellen, z.B. Spanisch) | - Im `PairingScreen`: Alle Texte (Titel, Label, Button, Fehler) werden in der Default-Sprache (Englisch) angezeigt.<br>- Im `LockScreen`: Die Nachricht wird in der Default-Sprache (Englisch) angezeigt.                                                                                                                                                                                 |

Diese Testszenarien decken die grundlegenden Interaktionen und Zustände der Kopplungsfunktion ab und helfen, eine positive User Experience sicherzustellen.
